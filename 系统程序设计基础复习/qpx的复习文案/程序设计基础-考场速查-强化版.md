## 1. 第 1、2 章超短回顾

### 1.1 C 程序与编译

```c
#include <stdio.h>
int main(void) {
    return 0;
}
```

编译链：

| 阶段  | 常用命令                | 结果    |
| --- | ------------------- | ----- |
| 预处理 | `gcc -E a.c -o a.i` | `.i`  |
| 编译  | `gcc -S a.i -o a.s` | `.s`  |
| 汇编  | `gcc -c a.s -o a.o` | `.o`  |
| 链接  | `gcc a.o -o app`    | 可执行文件 |
|     |                     |       |

常用选项：

- `-g`：生成调试信息
- `-Wall -Wextra`：打开常用警告
- `-O0/-O1/-O2/-O3/-Og/-Os/-Ofast`：优化级别

有关`O`：
- `-O0`：默认，不优化，最好调试
- `-O1` / `-O2` / `-O3`：优化逐渐增强
- `-Ofast`：更激进

#### GCC常见选项：
- `-E`：只预处理
- `-S`：编译成汇编，不汇编
- `-c`：生成目标文件，不链接
- `-o`：指定输出名
- `-m32` / `-m64`：生成 32/64 位代码
- `-g`：带调试信息

### 1.2 GDB 最小集合

- `b file:line`：下断点
- `r`：运行
- `p x`：打印变量
- `bt`：看调用栈

### 1.3 空指针

- `NULL` 表示空，不指向可访问对象
- 常见判断：`if (p)`、`if (!p)`

## 2. 第 5 章 指针的高级应用

### 2.1 指针声明与优先级

```c
int (*ptr)[5];     // 指向长度为5的int数组
int *parr[5];      // 长度为5的指针数组
int **ip;          // 二级指针
int *fip();        // 返回int*的函数
int (*pti)();      // 指向函数的指针
int *(*pfpi)();    // 指向“返回int*的函数”的指针
```

口诀：

- `[]`、`()` 优先级高于 `*`
- 看不懂就“从变量名出发，向外一层层读”

### 2.2 指针运算规则

- `&`：取地址
- `*`：解引用
- `p + n`：移动 `n * sizeof(*p)` 个字节
- `p - q`：仅当 `p`、`q` 指向同一数组中的元素时才有意义
- `p[i] == *(p + i)`

只允许这样比：

- 同类型指针可比 ` == ` / ` != `
- 指向同一数组内元素时可比 `< <= > >=`
- 任意指针都可与 `NULL` 比较

### 2.3 常见危险指针

| 类型 | 含义 | 后果 |
| --- | --- | --- |
| 野指针 | 未初始化或越界后得到的无效指针 | 结果不确定 |
| 悬空指针 | 指向已释放内存 | `use-after-free` |
| 空指针 | `NULL` | 解引用会崩 |
| `void *` | 通用指针 | 使用前需转型 |

安全原则：

- 定义即初始化：`int *p = NULL;`
- 用前判空
- `free(p); p = NULL;`
- 不乱做强制类型转换

### 2.4 `const` 指针

```c
const int *p;   // 不能改 *p，可以改 p
int *const p;   // 不能改 p，可以改 *p
```

### 2.5 数组、数组指针、指针数组

核心区别：

- 数组名是“首元素地址常量”，本身不能被赋值
- `int (*pa)[4]`：`pa` 指向“一整行”
- `int *pa[4]`：`pa` 是 4 个 `int*`

二维数组传参常用两种：

```c
void f1(int *p, int n);
void f2(int (*p)[4], int m, int n);
```

### 2.6 二级/多级指针

常见用途：

- 修改外层指针本身
- 构造动态二维/三维数组
- 处理“链表的链表”等复杂结构

动态二维数组两种套路：

1. 一整块连续内存

```c
int (*a)[10] = malloc(rows * 10 * sizeof(int));
```

2. 行指针 + 每行单独分配

```c
int **a = malloc(rows * sizeof(int *));
for (int i = 0; i < rows; i++) a[i] = malloc(cols * sizeof(int));
```

释放顺序：

- 第二种必须先逐行 `free(a[i])`，最后 `free(a)`

### 2.7 带指针的函数与函数指针

函数指针定义：

```c
double (*p)(double);
```

调用：

```c
p = square;
s = p(1.6);
```

用途：

- 回调
- 通用排序/查找
- 动态选择算法

### 2.8 `qsort` / `bsearch`

```c
void qsort(void *base, size_t n, size_t size,
           int (*cmp)(const void *, const void *));

void *bsearch(const void *key, const void *base,
              size_t n, size_t size,
              int (*cmp)(const void *, const void *));
```

比较函数规范：

- 返回 `< 0`：前者应在前
- 返回 `0`：相等
- 返回 `> 0`：后者应在前

常见写法：

```c
int cmp_int(const void *a, const void *b) {
    return *(const int *)a - *(const int *)b;
}
```

注意：

- `bsearch` 前提：数组已按同一比较规则有序

### 2.9 动态内存分配

```c
void *malloc(size_t size);
void *calloc(size_t n, size_t size);
void *realloc(void *ptr, size_t size);
void free(void *ptr);
```

差异：

| 函数 | 特点 |
| --- | --- |
| `malloc` | 分配但不清零 |
| `calloc` | 分配并清零 |
| `realloc` | 调整已有块大小 |
| `free` | 释放堆内存 |

`realloc` 三种情况：

1. 原地扩容
2. 异地扩容并拷贝旧数据
3. 缩小块大小

高频错点：

- 只能 `free` 动态分配来的内存
- 丢失首地址会造成内存泄漏
- `realloc` 后旧指针可能失效

### 2.10 柔性数组

```c
typedef struct {
    int length;
    int items[];
} Node;
Node *p = malloc(sizeof(Node) + n * sizeof(int));
```

与指针成员比较：

| 项目 | 柔性数组 | 指针成员 |
| --- | --- | --- |
| 分配次数 | 1 次 | 2 次 |
| 释放次数 | 1 次 | 2 次 |
| 连续性 | 连续 | 不连续 |
| 访问效率 | 高 | 较低 |

限制：

- 必须放最后
- 不能直接定义这种结构体数组

### 2.11 内存池

思想：

- 预先申请一大块连续内存
- 自己维护空闲块链表/索引

优缺点：

| 维度    | 内存池         | `malloc` |
| ----- | ----------- | -------- |
| 分配速度  | 快，常见 `O(1)` | 较慢       |
| 碎片    | 少           | 容易碎片化    |
| 管理复杂度 | 高           | 低        |

适用：

- 高频、小块、固定大小对象分配

### 2.12 链表

单链表节点：

```c
struct Node {
    int data;
    struct Node *next;
};
```

高频操作：

- 头插：新节点在前
- 尾插：保持尾指针
- 删除：找到待删节点及前驱
- 整表销毁：从头依次释放

高频结构变体：

- 虚拟头结点：统一“头部插入”和“中间插入”
- 双向链表：方便找前驱
- 循环链表：尾结点回指头

### 2.13 内存调试

五大内存 Bug：

1. 内存泄漏
2. 野/悬空指针
3. 缓冲区溢出
4. 非法访问
5. 重复释放

工具定位：

| 工具              | 强项                     |
| --------------- | ---------------------- |
| `-Werror`       | 将错误当作警告                |
| `-Wall -Wextra` | 编译期发现明显错误              |
| Cppcheck        | 静态逻辑漏洞                 |
| GDB             | 崩溃现场                   |
| Valgrind        | 泄漏、非法读写、未初始化值          |
| ASan            | 堆/栈越界、`use-after-free` |

>Cppcheck 的定位：在通过编译之后，继续做更深的静态逻辑分析

擅长发现：
- 提前返回路径上的泄漏
- 先解引用后判空
- 静态数组越界
- 释放后再使用

>Valgrind：

- 模拟执行
- 查得很细
- 但很慢

>ASan：

- 编译器内置
- 运行较快
- 日常开发很实用

常用命令：

```bash
gcc -Wall -Wextra -Werror main.c -o main
cppcheck --enable=all --inconclusive test.c
valgrind --leak-check=full ./app
gcc -fsanitize=address -g test.c -o app
```

#### 分阶段调试思路：

- 开发阶段：静态分析 + 安全写法
- 测试阶段：Valgrind / ASan / GDB
- 发布阶段：关注性能与缓存

### 2.14 内存优化

关注点：
- 内存布局是否高效
- CPU 缓存是否友好
- 分配策略是否合理

#### 结构体对齐与重排

规则：

1. 结构体首地址满足对齐要求
2. 成员偏移量满足自身对齐
3. 整体大小是最大对齐单位的整数倍

目的：让CPU访问更高效

优化原则：

- 大成员在前，小成员在后

#### 位域

优势在于：节省空间

```c
struct Flags {
    unsigned int active : 1;
    unsigned int priority : 3;
};
```

注意：

- 不能对位域取地址
- 不能用指针指向位域
- 位域数组不允许
- 溢出行为和现实有关

#### 联合体

```c
union Data {
    int i;
    double d;
};
// 加上标签就可以构成简单的变体记录
```

特点：

- 同一块内存，多种解释方式
- 适合“任一时刻只用一个成员”的场景

#### `memset` / `memcpy`

```c
memset(a, 0, sizeof(a));
memcpy(dst, src, sizeof(src));
```

易错：

- `memset(a, 1, sizeof(a))` 不是把 `int` 数组设成 1，而是把每个字节都设成 `0x01`，这是一种按照字节进行清零的方式
- `memcpy` 第三个参数是字节数，不是元素数，适合连续内存块复制
- 源目标重叠时不能用 `memcpy`，要用 `memmove`

#### 缓存友好

- 二维数组按行访问比按列访问快
- 连续数组通常比链表更缓存友好
- 小而短命的数据优先放栈上
- 延迟分配与对象重用
- 高频对象考虑内存池

## 3. 第 6 章 文件操作与数据持久化

### 3.1 文件模式

| 模式             | 含义          |
| -------------- | ----------- |
| `r`            | 只读，文件必须存在   |
| `w`            | 只写，若存在则清空   |
| `a`            | 追加写，写位置在文件尾 |
| `r+`           | 读写，文件必须存在   |
| `w+`           | 读写，创建或清空    |
| `a+`           | 读写，写总是在末尾   |
| `rb`/`wb`/`ab` | 二进制版本       |

### 3.2 基本文件操作

```c
FILE *fopen(const char *name, const char *mode);
int fclose(FILE *fp);
FILE *freopen(const char *name, const char *mode, FILE *fp);
```

标准流：

- `stdin`
- `stdout`
- `stderr`

### 3.3 文本 / 二进制 I/O

文本文件：

- 用人可读字符保存
- 便于查看和调试
- 但解析成本更高

二进制文件：

- 直接按字节保存内存表示
- 更省空间
- 读写效率更高
- 但不直观，对布局很敏感

总结：

- 文本更通用
- 二进制更接近内存

文本：

```c
fscanf(fp, "%d", &x);
fprintf(fp, "%d\n", x);
fgets(buf, n, fp);
fputs(buf, fp);
```

1. 格式化读写：`fscanf` / `fprintf`
2. 行读写：`fgets` / `fputs`
3. 块读写：`fread` / `fwrite`
4. 字节读写：`fgetc` / `fputc`

二进制块：

```c
size_t fread(void *buf, size_t size, size_t n, FILE *fp);
size_t fwrite(const void *buf, size_t size, size_t n, FILE *fp);
```

参数：
- `size`：每个元素大小
- `n`：元素个数

字节：

```c
int fgetc(FILE *fp);
int fputc(int c, FILE *fp);
```

### 3.4 为什么 `fgetc` 返回 `int`

- 要区分合法字节值和 `EOF`
- `EOF` 常为 `-1`
- 若用 `char` 接收，可能无法区分 `0xFF` 和 `EOF`

标准循环：

```c
int c;
while ((c = fgetc(fp)) != EOF) {
    ...
}
```

错法：

```c
while (!feof(fp)) { ... }   // 错
```

- `feof(fp)` 只有在一次读取失败并越过文件尾之后才会变真

>也就是说，在这里`feof(fp)`判断上一次读操作是否已经因为到达 EOF 而失败，具有滞后性

### 3.5 序列化与反序列化

定义：

- 序列化：内存对象 -> 连续字节流
- 反序列化：字节流 -> 内存对象

数组协议：

1. 先写长度
2. 再写所有元素

简单结构体可直接 `fwrite/fread` 的条件：

- 不含指针成员
- 内存布局可直接持久化

### 3.6 对齐与紧凑序列化

问题：

- 直接写结构体会把填充字节也写进去
- 换机器/换对齐规则可能错位

做法：

```c
#pragma pack(push, 1)
typedef struct { ... } PackedStudent;
#pragma pack(pop)
```

原则：

- 内存中用自然对齐结构体
- 落盘/传输时用紧凑结构体或明确协议

### 3.7 浅拷贝 vs 深拷贝

含指针成员时，不能直接整块写结构体。

错误：

```c
fwrite(&s, sizeof(Student), 1, fp);   // 只把地址写进文件
```

正确协议：

1. 写字符串长度
2. 写字符串内容
3. 写其他标量字段

### 3.8 链表序列化

不能只写头结点。

正确流程：

1. 先统计结点数 `N`
2. 先写 `N`
3. 逐个写数据域
4. 反序列化时 `malloc` 新结点并重新连接

### 3.9 魔数与版本号

文件头建议：

```c
typedef struct {
    uint32_t magic;
    uint16_t version;
} FileHeader;
```

意义：

- 验证格式合法性
- 拒绝错误文件
- 支持协议升级

### 3.10 文件定位与随机访问

```c
int fseek(FILE *fp, long offset, int origin);
long ftell(FILE *fp);
void rewind(FILE *fp);
int fgetpos(FILE *fp, fpos_t *pos);
int fsetpos(FILE *fp, const fpos_t *pos);
```

参考点：

- `SEEK_SET`
- `SEEK_CUR`
- `SEEK_END`

随机访问定长记录公式：

```c
offset = index * sizeof(Record);
```

### 3.11 更新模式读写切换

若以 `r+ / w+ / a+` 打开：

- 读后立刻写，或写后立刻读
- 中间必须插 `fseek(...)` 或 `fflush(...)`

即使：

```c
fseek(fp, 0, SEEK_CUR);
```

也能起到重置流属性的作用。

### 3.12 删除与索引

文本文件删除

：

- 本质上通常要“重写整个文件”

二进制定长记录删除：

- 常用逻辑删除：加 `isDeleted`
- 删除时把标志位置 1
- 读取时跳过

索引文件适用：

- ID 不连续
- 记录不定长
- 无法直接用数学公式算偏移

索引项典型格式：

```c
typedef struct {
    int id;
    long offset;
} IndexItem;
```

### 3.13 错误处理

```c
if (!fp) perror("open failed");
```

常用：

- `feof(fp)`：是否到文件尾
- `ferror(fp)`：最近一次 I/O 是否出错
- `clearerr(fp)`：清除 `EOF` 和错误标志
- `errno`
- `strerror(errno)`
- `perror(msg)`

判断循环结束原因：

```c
if (ferror(fp)) { ... }
else if (feof(fp)) { ... }
```

### 3.14 缓冲区、临时文件、删除重命名

```c
fflush(fp);
setvbuf(fp, NULL, _IONBF, 0);
FILE *tmp = tmpfile();
remove("a.txt");
rename("old.txt", "new.txt");
```

缓冲模式：

| 模式 | 含义 |
| --- | --- |
| `_IOFBF` | 全缓冲 |
| `_IOLBF` | 行缓冲 |
| `_IONBF` | 无缓冲 |

资源泄漏警告：

- `fopen` 后一定 `fclose`
- 循环内开文件尤其危险

## 4. 第 7 章A 多线程程序设计

### 4.1 基本概念

- 线程：进程内的一条执行流
- 并发：多个任务“同时推进”
- 并行：多个任务“同时执行”

线程共享：

- 代码段
- 全局/静态数据
- 堆
- 打开的文件等资源

线程私有：

- 栈
- 寄存器
- 程序计数器

### 4.2 最小 `pthread`

线程函数签名：

```c
void *thread_func(void *arg);
```

创建线程：

```c
pthread_t t;
pthread_create(&t, NULL, thread_func, &arg);
```

编译：

```bash
gcc main.c -lpthread
```

### 4.3 `pthread_create` / `pthread_exit`

```c
int pthread_create(pthread_t *tid, const pthread_attr_t *attr,
                   void *(*start)(void *), void *arg);
void pthread_exit(void *retval);
```

四个参数：

1. 线程 ID 地址
2. 属性，入门常用 `NULL`
3. 线程函数
4. 传入参数

主线程末尾：

- `return 0;`：整个进程结束，子线程也没了
- `pthread_exit(NULL);`：主线程自己结束，进程等其他线程结束

### 4.4 竞态条件

`counter = counter + 1` 在底层不是原子操作，而是：

1. 读
2. 改
3. 写

两个线程交错执行就可能“丢更新”，结果小于理论值。

### 4.5 忙等待

共享标志位 + `while` 轮询：

- 浪费 CPU
- 同步不可靠

结论：不推荐。

### 4.6 互斥锁

```c
pthread_mutex_t m = PTHREAD_MUTEX_INITIALIZER;
pthread_mutex_lock(&m);
pthread_mutex_unlock(&m);
```

完整 API：

```c
pthread_mutex_init(...)
pthread_mutex_destroy(...)
pthread_mutex_lock(...)
pthread_mutex_unlock(...)
```

关键规则：

- 成对出现
- 谁加锁谁解锁
- 锁保护“关键区”

### 4.7 死锁

典型原因：

- 两线程拿两把锁，顺序不一致

规避：

- 统一加锁顺序

### 4.8 条件变量

作用：

- 条件不满足时睡眠等待
- 条件满足时被唤醒
- 避免 Busy Wait

与互斥锁配合：

- 互斥锁负责“谁能进关键区”
- 条件变量负责“什么时候继续干活”

### 4.9 `join` / `detach`

```c
pthread_join(t, NULL);
pthread_detach(t);
```

区别：

| 函数 | 含义 |
| --- | --- |
| `pthread_join` | 等待指定线程结束并回收 |
| `pthread_detach` | 线程后台运行，结束后自动回收 |

### 4.10 题型思路

并发下载模拟：

- 串行总时间 = 各任务耗时之和
- 并发总时间 ≈ 最慢任务耗时

并行求和：

- 划分区间
- 每线程只处理自己那一段
- 主线程 `join` 后汇总

## 5. 第 7 章B 高级编程与性能优化

### 5.1 位运算

```c
a & b   // 与
a | b   // 或
a ^ b   // 异或
~a      // 取反
a << n  // 左移
a >> n  // 右移
```

高频技巧：

- 取第 `k` 位：`(x >> k) & 1`
- 置 1：`x |= (1 << k)`
- 清 0：`x &= ~(1 << k)`
- 翻转：`x ^= (1 << k)`
- `x % 2^n == x & (2^n - 1)`，只对非负数直观可靠

### 5.2 宏

基本形式：

```c
#define AREA(W, H) ((W) * (H))
```

高频坑：

1. 参数和整体不加括号
2. 多语句宏直接展开到 `if` 后出错
3. 带副作用实参被重复求值

推荐多语句宏：

```c
#define SWAP(a, b) do { int t = (a); (a) = (b); (b) = t; } while (0)
```

特殊操作：

- `#x`：字符串化
- `a##b`：符号拼接

### 5.3 条件编译与系统宏

```c
#ifdef DEBUG
...
#endif
```

常见系统宏：

- `__FILE__`
- `__LINE__`
- `__DATE__`
- `__TIME__`
- `__func__`
- `__GNUC__`
- `_WIN32`
- `__linux__`

### 5.4 `assert`

```c
#include <assert.h>
assert(ptr != NULL);
```

特点：

- 调试时用于“快速发现假设不成立”
- 定义 `NDEBUG` 后会被禁用

### 5.5 静态库与动态库

| 类型 | 特点 |
| --- | --- |
| 静态库 | 链接时拷进可执行文件 |
| 动态库 | 运行时加载，共享一份库副本 |

静态库：

```bash
gcc -c math_utils.c -o math_utils.o
ar rcs libmymath.a math_utils.o
gcc main.c -L. -lmymath -o myapp
```

动态库：

```bash
gcc -fPIC -c math_utils.c -o math_utils.o
gcc -shared -o libmymath.so math_utils.o
gcc main.c -L. -lmymath -o myapp
```

动态库查找：

- `-Wl,-rpath=...`
- `LD_LIBRARY_PATH`
- 系统库目录

查看导出符号：

```bash
nm -D libmymath.so
```

```cmake
add_library(my_static STATIC src/math_lib.c)

```

### 5.7 优化思路总序

1. 先找瓶颈
2. 先改算法和数据结构
3. 再改分支/循环/访问模式
4. 最后才考虑位运算和编译选项

### 5.8 函数级优化

- `inline`
- 尾递归优化
- 大结构体按指针传递
- `restrict` 消除别名
- `static` / `const`
- `__attribute__((const))`
- `__attribute__((pure))`
- `__attribute__((noreturn))`

`restrict` 含义：

- 告诉编译器几个指针不重叠，便于向量化

### 5.9 控制流优化

- 大概率分支放前面
- 异常路径用卫语句提前返回
- `LIKELY/UNLIKELY`
- 查表法代替分支
- 三元运算符/位运算实现无分支

### 5.10 循环优化

- 展开：减少判断次数
- 融合：两次遍历变一次
- 外提：不变量放循环外
- 交换：连续内存访问放内层
- 终点判断：可递减到 0
- 哨兵位：把越界检查和命中检查合并

### 5.11 运算级优化

- `x * 2^n -> x << n`
- `x / 2^n -> x >> n`
- `x % 2^n -> x & (2^n - 1)`
- 多次除同一常数，可预存倒数改乘法
- 比较长度时比较平方值，避免 `sqrt`
- 浮点慢且可能不精确，场景允许时可用定点数

### 5.12 性能分析工具

| 工具 | 特点 |
| --- | --- |
| `gprof` | 插桩，适合看函数调用频率 |
| `perf` | 采样，能看更底层性能热点 |

命令：


## 6. 高频易错清单

1. `int *p; *p = 1;`：未初始化指针。
2. `free(&x);`：释放了栈变量。
3. `free(p); free(p);`：重复释放。
4. `return &local;`：返回局部变量地址。
5. `realloc` 后继续用旧地址。
6. 二维动态数组只 `free(matrix)`，忘了逐行释放。
7. `memset(int_array, 1, sizeof(int_array))` 误以为都赋值为 1。
8. `memcpy` 处理重叠区间。
9. `while (!feof(fp))`。
10. `char c = fgetc(fp);`。
11. `fwrite` 直接写含指针的结构体。
12. 读写模式下“读完立刻写”却没 `fseek/fflush`。
13. 用 `"wb"` 打开后想原地修改，结果文件被清空。
14. 文本文件上滥用 `fseek`。
15. 主线程 `return 0` 导致子线程提前结束。
16. 多线程共享变量未加锁。
17. 两把锁加锁顺序不一致导致死锁。
18. 宏没加括号：`AREA(a+1, b+1)` 展开错误。
19. 宏参数带 `++` 导致重复求值。
20. 盲目上 `-O3`，没测量就宣称“更快”。

## 7. 考场模板句

### 7.1 解释竞态条件

“`counter++` 不是原子操作，会被拆成读、改、写三步；两个线程交错执行时可能发生丢失更新，因此结果小于理论值，需用互斥锁保护关键区。”

### 7.2 解释为什么不能直接序列化带指针的结构体

“因为 `fwrite` 只会把指针值本身写入文件，保存的是当前进程地址空间中的地址，不是指针指向的数据内容；重新加载后该地址无效，因此必须采用深拷贝协议。”

### 7.3 解释为什么 `fgetc` 返回 `int`

“因为需要额外表示 `EOF`，若返回 `char`，则可能无法区分合法字节值和 `EOF`。”

### 7.4 解释结构体重排优化

“调整成员顺序可减少对齐填充字节，从而减小结构体大小，尤其在结构体数组中可显著节省空间并提高缓存利用率。”

### 7.5 解释 `qsort` 的比较函数

“`qsort` 不知道元素真实类型，只把待比较元素地址传给用户提供的比较函数，再根据返回值决定顺序，因此比较函数本质是回调函数。”
